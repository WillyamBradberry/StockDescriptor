# SPEC-006: Resize & Navigation Scripts

**Status:** ✅ Done  
**Priority:** P1  
**Key Files:** `processing/resize_for_vision.ps1`, `processing/create-metadata-nav-modified.ps1`  
**Dependencies:** SPEC-003 (Metadata Pipeline)

---

## 1. Purpose

Provide PowerShell utility scripts for two auxiliary pipeline steps:
1. **Resize** — create downscaled JPG thumbnails for Vision API (reduces bandwidth/API cost)
2. **Navigation** — generate an Obsidian-compatible navigation page (METADATA-NAV.md) with copy/paste blocks

---

## 2. Resize Script (`resize_for_vision.ps1`)

### 2.1 Invocation

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File resize_for_vision.ps1 -ImageFolder "C:\path\to\images"
```

### 2.2 Behavior

1. Scan `{ImageFolder}/*.jpg` for original images
2. Create `{ImageFolder}/THMBS/` subfolder if not exists
3. For each JPG:
   - Proportionally resize to **1024px** on the longest side
   - Save to `THMBS/{filename}.jpg`
   - Preserve aspect ratio
4. Skip files that already exist in THMBS (incremental)

### 2.3 Edge Cases

- **No JPGs found** → warning, continue
- **Corrupted JPG** → skip with error log
- **THMBS creation fails** → abort with error
- **Already processed** → skip (no re-processing)

---

## 3. Navigation Script (`create-metadata-nav-modified.ps1`)

### 3.1 Invocation

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File create-metadata-nav-modified.ps1 -MetadataFile "C:\path\to\METADATA.md"
```

### 3.2 Behavior

1. Read `METADATA.md` (generated by SPEC-003)
2. Generate `METADATA-NAV.md` in the same folder
3. Navigation format:
   - List of all images with thumbnails
   - Copy/paste blocks for Title, Description, Keywords
   - Obsidian-compatible links (`![[filename.jpg]]`)
4. Output file is ready for Obsidian vault browsing

---

## 4. Pipeline Integration

Both scripts are called automatically by the pipeline:

| Step | Script | Called From |
|------|--------|-------------|
| 1/4 | `resize_for_vision.ps1` | `gui_launcher.py:_pipeline_worker()` |
| 4/4 | `create-metadata-nav-modified.ps1` | `batch_metadata.run_nav_script()` |

---

## 5. Error Handling

- **Script not found** → step skipped with warning (non-fatal)
- **Script execution fails** → error logged, pipeline continues
- **METADATA.md missing** → nav script skipped
