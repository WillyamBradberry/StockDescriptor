# SPEC-004: EXIF Injection

**Status:** ✅ Done  
**Priority:** P0  
**Key File:** `processing/write_exif.ps1`  
**Dependencies:** SPEC-003 (Metadata Pipeline)

---

## 1. Purpose

Inject AI-generated metadata (Title, Description, Keywords) from METADATA.md into the original JPG files as EXIF/IPTC fields, making the metadata permanently embedded in the image files for stock platform submission.

---

## 2. Input / Output

### 2.1 Input

| Source | Description |
|--------|-------------|
| `METADATA.md` | Generated by SPEC-003 — contains Title/Description/Keywords per filename |
| `{image_folder}/*.jpg` | Original JPG files to inject metadata into |
| `-ExifToolPath` | Optional: path to exiftool.exe |

### 2.2 Output

- Original JPG files modified in-place with new EXIF/IPTC metadata
- No new files created

---

## 3. Fields Injected

| Field | EXIF/IPTC Standard | Source from METADATA.md |
|-------|-------------------|------------------------|
| Title | IPTC:ObjectName | `**Title:**` block |
| Description | IPTC:Caption-Abstract | `**Description:**` block |
| Keywords | IPTC:Keywords | `**Keywords:**` block (parsed comma-separated) |

---

## 4. Implementation (PowerShell + ExifTool)

### 4.1 Invocation

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File write_exif.ps1 -OriginalFolder "C:\path\to\images" [-ExifToolPath "D:\exiftool.exe"]
```

### 4.2 Algorithm

1. Scan `{OriginalFolder}/*.jpg`
2. For each JPG, look up its metadata in METADATA.md
3. Build ExifTool arguments:
   ```
   exiftool.exe
     -overwrite_original
     -IPTC:ObjectName="{title}"
     -IPTC:Caption-Abstract="{description}"
     -IPTC:Keywords="{keyword1}"
     -IPTC:Keywords="{keyword2}"
     ...
     "{jpg_path}"
   ```
4. Execute ExifTool for each file

### 4.3 ExifTool Path Resolution

1. Use `-ExifToolPath` argument if provided
2. Fallback: lookup via `config_manager.py` (`exiftool_path` field)
3. Default: `D:\PROGRAMS\EXIFTOOL\exiftool.exe`

---

## 5. Error Handling

- **Missing METADATA.md** → abort with error
- **ExifTool not found** → abort with error (path must be configured)
- **File write error** → log and continue (non-fatal for batch)
- **JPG not in METADATA.md** → skip with warning

---

## 6. Integration

### 6.1 Pipeline Integration

Called automatically by `batch_metadata.py` after metadata generation, unless `--no-inject` flag is used.

### 6.2 Conditions

- **Skipped if** `metadata_error.md` exists (unresolved AI errors)
- **Skipped if** `write_exif.ps1` script not found
- **Skipped if** `--no-inject` flag passed
