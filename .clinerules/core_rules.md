# Project
All code, documentation, memory files,
reports, tasks and specifications
must exist inside project root directory.

Never create files outside project root.

Preserve existing functionality.

Never create placeholder implementations.

Memory Bank is the source of truth
for project knowledge and decisions.

Whenever any of the following occurs:
    * Architecture changes
    * Folder structure changes
    * New design patterns appear
    * Important discoveries are made
    * Refactoring decisions are taken
    * Technical limitations are found
    * Major tasks are completed
You MUST update the appropriate Memory Bank files.

Determine task type before work:
from .\docs\rules
- BUGFIX: load debugging.md
- FEATURE: Load implementation.md
- REFACTOR: load refactoring.md
- REVIEW: load review.md
